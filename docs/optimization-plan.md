## 阶段二：性能优化 (P1)

**目标**: 提升响应速度，减少内存占用

### 2.1 搜索防抖优化
- [ ] 在 popup.js 搜索输入添加防抖 (300ms)
- [ ] 防抖期间显示加载指示器
- [ ] 取消未完成的搜索请求

### 2.2 渲染性能优化
- [ ] 合并 `renderFolders()` 和 `filterAndRender()` 调用
- [ ] 使用 `requestAnimationFrame` 优化渲染
- [ ] 大量书签时启用虚拟滚动 (>100 条)

### 2.3 缓存策略优化
- [ ] 书签变化时增量更新缓存 (非全量重建)
- [ ] 搜索索引持久化到 `chrome.storage.local`
- [ ] 添加缓存过期机制

### 2.4 Service Worker 优化
- [ ] 替换 `setInterval` 为 `chrome.alarms` API
- [ ] 减少不必要的保活操作

**验收标准**:
- 搜索响应时间 < 100ms (1000 书签)
- 首屏渲染时间 < 200ms
- 内存占用降低 20%

## 阶段三：代码重构 (P2)

**目标**: 提高代码可维护性

### 3.1 拆分 popup.js
```
src/popup/
├── popup.js          # 主入口，初始化
├── modules/
│   ├── i18n.js       # 国际化模块
│   ├── search.js     # 搜索逻辑
│   ├── render.js     # 渲染逻辑
│   ├── folder.js     # 文件夹操作
│   ├── bookmark.js   # 书签操作
│   └── utils.js      # 工具函数
```

### 3.2 提取常量
- [ ] 创建 `src/constants.js`
- [ ] 搜索评分权重常量化
- [ ] 颜色配置常量化
- [ ] 系统文件夹 ID 常量化

### 3.3 统一错误处理
- [ ] 创建 `src/utils/errorHandler.js`
- [ ] 统一 `sendMessage` 错误处理
- [ ] 添加全局错误捕获

### 3.4 消除重复代码
- [ ] 合并 `getFolderColor()` 和 `getFolderIconColor()`
- [ ] 提取公共的 DOM 操作函数
- [ ] 统一 Toast 和 Modal 调用方式

**验收标准**:
- 单文件代码量 < 500 行
- 无重复代码块
- ESLint 无警告
